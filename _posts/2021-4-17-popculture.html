# Kpop.

# Week 3: Relationships. Day 17: Pop Culture.


Day 17's topic was pop culture. Kpop has become super popular, and rightfully so, the amount of effort that goes into these kpop groups is so immense, groups train for years before they debut, the music videos are on point, the costumes are unique, so I definitely had to visualize kpop groups for day seventeen. I visualized a dendogram showing the relationship between kpop group member count and the change over the years. Here's what the dendogram looked like for the 90's period below.


Just like the chart above, respective dendograms were made for each period, i.e th 1990's, 2000's, 2010's and 2020's. Each period saw a significant number of five member groups but this five member group trend seems to have changed in 2020's, with seven member groups becoming more popular.

Using d3.hierarchy we create a hierarchial structure of the data, structuring the data in a hierarchy will help create the dendograms. Using d3.tree will give us position of the dendogram paths, d3.linkRadial will help make the dendogram circular.

const hierarchy = d3.hierarchy(data)

const tree = d3.tree()
                .size([2*Math.PI,220])

const root = tree(hierarchy)

const linkRadial = d3.linkRadial()
                    .angle(d=>d.x)
                    .radius(d=>d.y)

We have all the necessary functions ready to draw the elements of the chart. These are the steps below to draw the elements,

  1. Provide a chart title, lines 1 to 8.
  2. Draw the dendogram by appending a group element to the svg first, use transform attribute to draw the elements in the middle of the chart (line 10-11).
  3. Use d3.selectAll to provide a blank placeholder, bind the data with root.links() then join the path. line(12-13).
  4. Since the dendogram is a path, use attr "d" to call the linkRadial variable which has the d3.linkRadial() method to draw the cicular dendograms. line(15)
  5. Use prefered stroke colour and stroke-width, set fill to none.

Use the code below to annotate the various levels of the dendogram.

svg.append("g")
        .attr("transform",`translate(${width/2},${height/2+20})`)
        .attr("font-family","sans-serif")
        .attr("font-size",10)
        .attr("stroke-linejoin",3)
        .attr("stroke-width",3)
        .selectAll("text")
        .data(root.descendants())
        .join("text")
        .attr("transform",d=>`
            rotate(${d.x*180 / Math.PI-90})
            translate(${d.y},0)
            rotate(${d.x >=Math.PI?180:0})
            `)
        .attr("dy","0.31em")
        .attr("x",d=>d.x<Math.PI === !d.children? 6:-6)
        .style("fill","#f9ae91")
        .style("font-size","1.5em")
        .attr("text-anchor",d=>d.x<Math.PI === !d.children?"start":"end")
        .text(d=>d.depth==3?d.data.name:d.data[0])
        .clone(true)
        .lower()
        .attr("stroke","#15166d")
        .attr("stroke-width",5)

If you noticed above, I have not called the chart function for the 2010 group data. This is because the data for 2010 period is very large compared to the other periods, there has been a boom of new groups formed in the 2010 period. Calling the chart function we created above for 2010 data, creates the chart below.
The dendogram needs to be much bigger to clearly show the chart. To solve this problem create a separte chart function with a few changes, which will be used exculsively for 2010 data. This chart will also have a zoom feature to help the user easily read the chart. The 2010 final chart preview is below.

Most of the code for the 2010 chart above will remain the same as the previous chart function, there will only be a few additions and changes. Create the new chart function called "chart2".

chart2 = (data, title) => {}

The changes made in this function are detailed below.

  • Add a viewBox attribute the svg variable, as on line 6.
  • In the previous chart function, d3.tree had a size of 220, which is the radius of the dendogram, in this function, set the radius to 570 which will make the dendogram bigger and prevent overlapping of data (line 14).
  • The final change to this new chart function is tweaking the transform attribute on lines 34, 44. This will help center the dendogram to the middle of the svg.
